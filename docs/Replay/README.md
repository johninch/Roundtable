# 项目复盘

### 目标

推动 ESOP-FED 对已经完成的，有价值的，有挑战的项目进行复盘整理，得到真正的技术沉淀。

### 形式

1. 在「复盘项目看板」处积极挖坑，列出todo；
2. 当开发小组完成对应项目全部工作后，先由小组内部组织对项目进行复盘：
    - 总结出开发过程中遇到的难点，有价值的问题；
    - 自省做的好的或者不好的点；
    - 提炼技术实现关键点；
3. 写成复盘报告添加到Roundtable中Replay复盘中心（小组成员每人都需要写）；
4. 组织全员分享会，讨论和复盘，推动全组技术进步。
